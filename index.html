Event Details

"Every journey needs to get started."
"So here's AARAMBH."

  • The Computer Science and Engineering department of Shambhunath Institute of Engineering and Technology has came with the initiative of SCC (SIET Coding Club).
  • Aarambh is a coding competition held by SIET Coding Club (SCC) wherein you will compete with hundreds of coding enthusiasts. This competition will implant the quick problem solving skills into students.
  • This contest will consist of 4 DSA coding questions.
  • The time limit for the contest will be 120 minutes.
  • The top coders will be rewarded. Aarambh will be the first level up milestone of the SCC and the programmers to compete with the fellow coders with mutual interests in the Technical Arena.
  • Date and Time: 12 May 2022, 07:00 PM (IST)

FAQs

Where will the contest take place?

This contest is an online contest and will take place in Geeks for Geeks.

Is there any fee to participate in this contest?

No, it is free of cost.

How do I participate in this contest?

To participate in this challenge, do the following:
a. Register for the challenge
b. Visit the challenge page on start date/time
c. Click Participate in Challenge

Are the programming questions restricted to specific languages?

Yes, the programming questions are restricted to some languages. You can answer only in C++, Java and Python.

How does the online judge determine whether a solution is correct?

An online judge is a code checker and not a human being. The code checker or "judge" tests your code automatically.
Therefore, you must write your code accordingly. For each problem, based on the specifications mentioned in the problem statement, there will be one or more input files and corresponding (correct) output files.
The program that you create is run on each of the input files and the corresponding output must match the (correct) output files in order.
Your program will be marked incorrect in the following cases:
Output files (generated by running your code) do not match the pre-generated output files Output files (generated by running your code) matches all the pre-generated output files, however, it is not in the order in which the pre-generated output files are arranged.

How many times can I submit a solution?

You can make as many submissions as you want until the time runs out, the best submission will be considered for the evaluation but every wrong submission will led to 5% penalty.
